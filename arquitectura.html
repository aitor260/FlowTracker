Estructura y diseño¶

Peligro

La documentación, el diseño y la programación de este proyecto están en fase de desarrollo. Por lo tanto, nada de lo especificado es definitivo ni funcional.

Esta documentación recoge las especificaciones y decisiones de diseño que han dado forma al sistema en cada versión, sirviendo como referencia técnica para cualquier desarrollo presente o futuro.

Características de FlowTracker¶

FlowTracker v1.0 es una aplicación de escritorio desarrollada en Python 3.11 para la gestión contable personal. Opera de forma completamente local, sin dependencias externas, y organiza su código en tres capas bien diferenciadas.

  • Persistencia ligera: los datos se almacenan en ficheros JSON y CSV, manteniéndolos accesibles y auditables directamente.

  • Arquitectura modular: separación clara entre lógica de negocio (core), persistencia (data) e interfaz de usuario (ui).

  • Dominios independientes: cada dominio de core —account, entry, plan, project, record y report— se estructura en models.py y actions.py.

  • Interfaz dual: contempla CLI ya operativa y GUI en desarrollo, coordinadas a través de un controlador común.

Estructura¶

El proyecto se organiza en los siguientes directorios raíz:

flowtracker
├── doc
├── src
├── tests
├── README.md
├── pyproject.toml
└── requirements.txt

  • doc: documentación del proyecto.

  • src: código fuente de la aplicación.

  • tests: pruebas unitarias y de integración.

  • pyproject.toml: configuración del proyecto y dependencias.

  • requirements.txt: dependencias del entorno de desarrollo.

El código fuente se organiza en el directorio src/flowtracker, estructurado en tres capas principales.

src
└── flowtracker
    ├── __init__.py
    ├── __main__.py
    ├── core
    │   ├── account
    │   │   ├── actions.py
    │   │   └── models.py
    │   ├── entry
    │   │   ├── actions.py
    │   │   └── models.py
    │   ├── plan
    │   │   ├── actions.py
    │   │   └── models.py
    │   ├── project
    │   │   ├── actions.py
    │   │   └── models.py
    │   ├── record
    │   │   ├── actions.py
    │   │   └── models.py
    │   └── report
    │       ├── actions.py
    │       └── models.py

Nota

Los ficheros __init__.py se omiten del árbol por brevedad. Están presentes en todos los directorios del paquete.

Capa de lógica de negocio - core¶

Contiene toda la lógica de negocio de la aplicación, organizada en dominios independientes: account, entry, plan, project, record y report. Cada dominio sigue un patrón consistente: models.py define las estructuras de datos propias del dominio y actions.py implementa las operaciones sobre ellas. Esta separación facilita el testing unitario y el reemplazo independiente de cualquier componente.

Capa de persistencia - data¶

Gestiona el acceso y almacenamiento de datos en ficheros JSON y CSV. manager.py centraliza las operaciones de lectura y escritura, mientras que utils.py agrupa las utilidades de apoyo. El resto de la aplicación interactúa con los datos exclusivamente a través de esta capa.

Capa de interfaz de usuario - ui¶

Coordina la interacción con el usuario a través de controller.py, que actúa como punto de entrada común para las dos interfaces disponibles: la CLI, ya operativa, implementada en cli/, y la GUI en desarrollo, ubicada en gui/. Esta separación permite añadir o sustituir interfaces sin modificar la lógica de negocio subyacente.
